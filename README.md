## About Us

Hellstar is a Minecraft plugin development organization with a simple yet ambitious mission:
> **"Create premium-quality Minecraft plugins, but make them 100% free and open-source."**

We believe that server owners shouldn't have to pay dozens of dollars for well-optimized, secure, and feature-rich plugins. At Hellstar, "free" doesn't mean stripped features or cheap quality. We bring a new standard to open-source plugins.

## Contributing

We are highly open to community contributions! Below you can find detailed guidelines on how to get involved and help improve Hellstar plugins.

### Reporting Bugs and Problems
If you encounter any issues while using our plugins, please submit a bug report in the issues tab of the respective plugin repository:
*   Ensure you are using the latest version of the plugin and your server software (such as Paper, Purpur, or Folia).
*   Provide a clear and concise description of the problem.
*   Include steps to reproduce the issue.
*   Attach any relevant error logs or stack traces using a pasting service like pastebin.com or bytebin.co.

### Requesting Features
We welcome suggestions for new features and improvements:
*   Open an issue in the relevant repository using the feature request template if available.
*   Describe the feature you would like to see in detail and how it would benefit the community.
*   Provide potential use cases or configurations if applicable.

### Code Contributions
If you are a developer looking to contribute code:
1.  Fork the repository of the plugin you wish to modify.
2.  Create a new branch for your feature or bug fix (e.g., feature/your-feature-name or fix/bug-description).
3.  Ensure your code adheres to standard Java/Kotlin conventions (camelCase naming, clear documentation, etc.).
4.  Write clean, performance-optimized, and well-structured code.
5.  Test your changes thoroughly on a local server environment before submitting a Pull Request.
6.  Submit a Pull Request targeting the development or main branch as specified by the repository guidelines.

### Translating and Localization
Help us make our plugins accessible to server owners worldwide:
*   Locate the language configuration file (such as messages.yml) in the plugin repository.
*   Translate the configuration keys into your target language.
*   Submit a Pull Request with the translated file (e.g., messages_de.yml for German or messages_fr.yml for French).
